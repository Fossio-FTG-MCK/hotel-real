Parâmetros da URL de Retorno - InfinitePay

Objetivo

Esta página descreve os parâmetros que são enviados para o Hotel Real após a confirmação de pagamento pelo sistema InfinitePay. A URL de retorno é utilizada para registrar automaticamente a reserva e atribuir pontos ao cliente.

Parâmetros esperados na URL

Os seguintes dados são esperados como parâmetros da URL:

  • reserva_id – ID único da reserva (obrigatório)
  • suite_id – ID único da suite (obrigatório)
  • order_id – ID do pedido na InfinitePay (obrigatório)
  • nsu – Número da transação (obrigatório)
  • aut – Código de autorização do pagamento (obrigatório)
  • value – Valor total pago (obrigatório, formato 999.99)
  • nome – Nome do cliente (obrigatório)
  • email – E-mail do cliente (obrigatório)
  • whats – WhatsApp do cliente (opcional)
  • checkin – Data de check-in (opcional)
  • checkout – Data de check-out (opcional)

Exemplo de URL completa

https://seusite.com/obrigado.html?reserva_id=reserva_20250420_12345&suite_id=0320059e-f8a4-4581-9863-22cbe071292f&order_id=1234&nsu=ABC123&aut=999888&value=750.00&nome=João%20Silva&email=joao@example.com&whats=5511999999999&checkin=2025-04-20&checkout=2025-04-23

Importância dos dados

Os parâmetros obrigatórios são essenciais para registrar a transação com segurança e evitar duplicações. Os dados opcionais enriquecem o histórico da reserva e ajudam no atendimento ao cliente.
